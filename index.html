JSchema

An experiment in simplicity

See also jschema-rpc

Introduction

One of the great advantages of JSON is the fact that it is so simple. Its grammar consists of fifteen easily understood productions, making it both easy to parse and produce.

A simple data interchange format demands a simple schema system. Unfortunately the current schema standard for JSON, JSON Schema is complicated, deriving many ideas from the XML-based schemas.

JSchema provides a simple way to specify the structure of JSON content. The design goals are:

  • Its grammar should be as simple as JSON itself
  • It should provide enough flexibility to encompass most existing JSON content
  • It should communicate the structure of the JSON document visually
  • It should map cleanly to standard programming language concepts and data structures

In order to retain simplicity it is necessary to forego some features. JSchema does not allow:

  • Named schema elements
  • Mixed content
  • Various useful data types (e.g. intervals).

Schema URL and File Conventions

Assuming JSON content matching a given JSchema is available at 

http://example.com/path/
then the JSchema should be available at 
http://example.com/path/?JSchema

Locally, JSchema files should end with the .jsc file suffix.

Grammar

<jschema_doc> ::= 
  <type>

<type> ::= 
  <core_types> |
  <struct_type> |
  <array_type> |
  <map_type> |
  <enum_type> |
  <ref_type>

<core_types> ::=
  "string" |
  "boolean" |
  "date" |
  "integer" |
  "decimal" |
  "biginteger" |
  "bigdecimal"

<struct_type> ::= 
  '{' <props> '}'

<props> ::= 
  prop |
  prop ',' props

<prop> ::= 
  <string> ':' <type>

<array_type> ::= 
  '[' type ']'

<map_type> ::=
  '{' '"map_of"' ':' '{' '"key"' ':' <type>, '"value"' ':' <type>  '}' '}'

<enum_type> ::= 
  '{' '"enum"' ':' '[' <enum_values> ']' '}'

<enum_values> ::=
  string
  string ',' <enum_values>

<ref_type> ::=
  '{' '"ref"' ':' string '}'

Types

"string"

A string may have a value of a standard JSON string.

"boolean"

A boolean may have a value of either the JSON String "true" or the JSON string "false".

"date"

A date may have a value of a standard JSON string in any format specified by the W3C NOTE-datetime.

"integer"

An integer may be a JSON int representing any 32-bit signed integer.

"biginteger"

A biginteger may be a JSON int representing a signed integer of arbitrary size.

"decimal"

A decimal may be a JSON number representing any double precision 64-bit floating point number.

"bigdecimal"

A bigdecimal may be a JSON number representing any number of arbitrary size and precision.

Struct Type

A struct type defines a structure for a JSON object. Note that the returned JSON document need not include every field specified in the struct specification, but should not return any fields not specified in the json struct specification.

Array Type

An array type defines defines the values of a JSON Array.

Map Type

A map type defines defines the values of a JSON object. For each pair in the JSON object, the first value must satisfy the first type definition and the second value must satisfy the second type definition.

Enum Type

An enum must have a string value from domain specified.

Ref Type

A ref type is a reference to a jschema document. The second element is a URL that the jschema content can be retrieved from.

Examples

Schema Examples
{ "name" : "string", "age" : "integer" } { "name" : "Joe", "age" : "42" }
{ "people" : [ { "name" : "string", "age" : "integer"} ] } { "people" : [ { "name" : "Joe", "age" : "42" } { "name" : "Paul", "age" : "28" } { "name" : "Mack", "age" : "55" } ] }
{ "people" : [ { "name" : "string", "age" : "integer"} ] } { "people" : [ { "name" : "Joe", "age" : "42" }, { "name" : "Paul", "age" : "28" }, { "name" : "Mack", "age" : "55" } ] }
{ "people" : [ { "name" : "string", "age" : "integer", "eye_color" : {"enum" : ["brown", "blue", "green"]}} ] } { "people" : [ { "name" : "Joe", "age" : "42", "eye_color" : "brown" }, { "name" : "Paul", "age" : "28", "eye_color" : "brown" }, { "name" : "Mack", "age" : "55", "eye_color" : "blue" } ] }
{ "id_to_people" : { "map_of" : { "key" : "biginteger", "value" : { "name" : "string", "age" : "integer", "eye_color" : {"enum" : ["brown", "blue", "green"]} } } } } { "id_to_people" : { "1" : { "name" : "Joe", "age" : "42", "eye_color" : "brown" }, "2" : { "name" : "Paul", "age" : "28", "eye_color" : "brown" }, "3" : { "name" : "Mack", "age" : "55", "eye_color" : "blue" } } }
